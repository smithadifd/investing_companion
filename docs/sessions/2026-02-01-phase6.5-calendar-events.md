# Session: Phase 6.5 - Calendar & Events

**Date**: 2026-02-01
**Phase**: 6.5 Calendar & Events
**Status**: COMPLETE

---

## Overview

Implemented the calendar and economic events feature, enabling users to track earnings dates, FOMC meetings, CPI releases, and other macro events. Added per-watchlist-item calendar tracking control to avoid calendar clutter.

---

## What Was Accomplished

### Backend

1. **Economic Event Model & CRUD**
   - `EconomicEvent` model with event types: earnings, fomc, cpi, nfp, gdp, ex_dividend, stock_split
   - Full CRUD endpoints for events
   - Calendar month endpoint (`GET /events/calendar/{year}/{month}`)
   - Upcoming events endpoint with filters

2. **Earnings Calendar Service**
   - Integration with Yahoo Finance for earnings and dividend dates
   - `EconomicEventService` for fetching and storing equity events
   - Automatic refresh of watchlist events

3. **Macro Events Seeding**
   - Seed script for major 2026 macro events (FOMC, CPI, NFP, GDP)
   - Idempotent seeding (safe to run multiple times)

4. **Celery Task for Event Refresh**
   - Daily task to refresh earnings/dividend events for tracked equities
   - Rate limiting with 1.5s delay between Yahoo API calls
   - Filters by `track_calendar` flag on watchlist items

5. **Track Calendar Flag**
   - Added `track_calendar` boolean column to `watchlist_items`
   - Defaults to `true`, but existing items set to `false` except MSFT/NVDA
   - Migration: `20260201_006_add_track_calendar_to_watchlist_items`

### Frontend

1. **Calendar Page** (`/calendar`)
   - Month view with navigation
   - Event type filters (earnings, FOMC, CPI, etc.)
   - Watchlist-only toggle
   - Responsive grid layout

2. **Dashboard Widget**
   - `UpcomingEvents` component showing next 7 days of events
   - Quick navigation to full calendar

3. **Equity Detail Events**
   - `EquityEvents` component on equity detail pages
   - Shows upcoming earnings, dividends for the equity

4. **Watchlist Calendar Control**
   - Toggle in `EditItemModal` to enable/disable calendar tracking
   - Calendar icon indicator in `WatchlistItemRow` for tracked items

## Issues Encountered & Resolved

### 1. Path Parameter Error
**Issue**: Backend failed with `AssertionError: Cannot use Query for path param 'month'`
**Fix**: Changed `Query()` to `Path()` for year/month parameters in calendar endpoint

### 2. useAuth Import Error
**Issue**: Calendar page threw `TypeError: useAuth is not a function`
**Fix**: Changed import from `@/lib/hooks/useAuth` to `@/lib/contexts/AuthContext`

### 3. No Events Showing
**Issue**: Calendar showed no events after seeding
**Cause**: Seed script only created 2025 events, current date is 2026
**Fix**: Ran `python -m scripts.seed_macro_events --year 2026`

### 4. Database Log Error
**Issue**: Noticed `FATAL: database 'investing' does not exist` in DB logs
**Status**: Added to Phase 6.6 plan for investigation (likely a stray connection string)

## Database Changes

### New Table: `economic_events`
```sql
CREATE TABLE economic_events (
    id SERIAL PRIMARY KEY,
    event_type VARCHAR NOT NULL,  -- earnings, fomc, cpi, nfp, gdp, ex_dividend, stock_split
    equity_id INTEGER REFERENCES equities(id),  -- nullable for macro events
    event_date DATE NOT NULL,
    event_time TIME,
    title VARCHAR NOT NULL,
    description TEXT,
    actual_value VARCHAR,
    forecast_value VARCHAR,
    previous_value VARCHAR,
    importance VARCHAR DEFAULT 'medium',  -- low, medium, high
    source VARCHAR DEFAULT 'manual',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

### Modified Table: `watchlist_items`
```sql
ALTER TABLE watchlist_items
ADD COLUMN track_calendar BOOLEAN NOT NULL DEFAULT TRUE;
```
